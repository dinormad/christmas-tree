# 照片上传指南 / Photo Upload Guide

## 📸 如何使用照片上传功能

### 方法一：使用照片管理器（推荐）

1. 点击右下角的 **"📸 管理照片"** 按钮
2. 上传主照片（top.jpg）- 会显示在圣诞树顶的星星上
3. 上传树身照片 - 可以一次选择多张，会自动命名为 1.jpg, 2.jpg, 3.jpg...
4. 点击 **"查看手动保存说明"** 查看如何手动保存照片
5. 或点击 **"上传照片"** 尝试自动上传（需要后端支持）

### 方法二：手动添加照片

直接将照片文件放入项目的 `public/photos/` 文件夹：

```
public/photos/
  ├── top.jpg      # 顶端封面图（树顶星星）
  ├── 1.jpg        # 树身照片 1
  ├── 2.jpg        # 树身照片 2
  ├── 3.jpg        # 树身照片 3
  └── ...          # 更多照片
```

## 📋 照片要求

### 顶端封面图 (top.jpg)
- **用途**: 显示在圣诞树顶端的立体五角星上
- **命名**: 必须命名为 `top.jpg`
- **比例**: 建议使用正方形或 4:3 比例
- **大小**: 建议 500KB 以内

### 树身照片 (1.jpg, 2.jpg, 3.jpg...)
- **用途**: 以拍立得形式装饰圣诞树
- **命名**: 按顺序命名为 `1.jpg`, `2.jpg`, `3.jpg`...
- **比例**: 建议使用正方形或 4:3 比例
- **大小**: 单张建议 500KB 以内
- **数量**: 可以添加任意数量的照片

## 🎨 优化建议

1. **图片压缩**: 使用在线工具（如 TinyPNG）压缩图片以提高加载速度
2. **图片裁剪**: 确保照片中的主体居中，以便在拍立得框架中更好地显示
3. **图片格式**: 支持 JPG、PNG 等常见格式，建议使用 JPG 以获得更小的文件大小
4. **图片数量**: 虽然可以添加很多照片，但建议控制在 50 张以内以保证流畅度

## 🔧 开发环境说明

### 自动上传功能
当前项目为前端应用，自动上传功能需要后端 API 支持。在开发环境中：

1. 照片管理器会提供手动保存说明
2. 可以通过 "查看手动保存说明" 查看需要如何重命名文件
3. 手动将文件复制到 `public/photos/` 文件夹
4. 刷新页面即可看到更新的照片

### 生产环境部署
如果需要在生产环境中实现自动上传功能，需要：

1. 添加后端 API 端点 `/api/upload-photos`
2. 实现文件上传和保存逻辑
3. 配置 CORS 和文件大小限制
4. 可以参考 `api/upload-photos.ts` 中的示例代码

## 🌟 使用提示

- 每次添加新照片后，需要刷新浏览器页面才能看到更新
- 照片会随机分布在圣诞树上，每次刷新位置可能不同
- 使用手势控制（张开手掌/握拳）可以控制圣诞树的组合/散开状态
- 点击 "DEBUG" 按钮可以看到摄像头视图和手势识别状态

## ❓ 常见问题

**Q: 上传照片后看不到效果？**
A: 请刷新浏览器页面，并确保照片已正确保存到 `public/photos/` 文件夹。

**Q: 可以上传多少张照片？**
A: 理论上没有限制，但建议控制在 50 张以内以保证流畅度。

**Q: 支持什么格式的图片？**
A: 支持常见的图片格式如 JPG、PNG、WEBP 等。

**Q: 如何删除照片？**
A: 直接从 `public/photos/` 文件夹中删除对应的文件，然后刷新页面。
